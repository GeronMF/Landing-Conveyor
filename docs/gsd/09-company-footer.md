## Goal
Описать модуль информации о компании и футера: как настраиваются юридические и контактные данные, соцсети и pop-up страницы (политики), и как они переиспользуются на всех лендингах.

## Solution
Информация о компании и футере хранится в полях модели `Landing` (название компании, юридический текст, контакты, соцсети, копирайт, HTML‑содержимое политик) и редактируется на вкладке «Інфо компанії / Футер» в админке. На лендинге эта информация рендерится в нижней части страницы и в модальных окнах (для политик и условий возврата).

## Details

### Схема БД: поля футера и компании в `Landing`

Фрагмент `Landing`:

```text
model Landing {
  ...
  companyName        String?
  companyNameRu      String?
  legalText          String?       @db.Text
  legalTextRu        String?       @db.Text

  privacyPolicyText  String?       @db.LongText
  privacyPolicyTextRu String?      @db.LongText
  termsText          String?       @db.LongText
  termsTextRu        String?       @db.LongText

  copyrightText      String?
  copyrightTextRu    String?

  phone              String?
  email              String?
  socials            Json?
  links              Json?
  ...
}
```

`socials` обычно имеет структуру:

```json
{
  "facebook": "https://facebook.com/...",
  "instagram": "https://instagram.com/...",
  "youtube": "https://youtube.com/...",
  "tiktok": "https://tiktok.com/@..."
}
```

### Admin UI

На вкладке **«Інфо компанії / Футер»** (`/admin/landings/[id]`):

- **Название компании**:
  - `companyName` / `companyNameRu`.
- **Юридическая информация**:
  - `legalText` / `legalTextRu` — rich‑text редактор.
- **Контакты**:
  - `phone`, `email`.
- **Соцсети**:
  - поля для `facebook`, `instagram`, `youtube`, `tiktok` и др. через JSON‑объект `socials`.
- **Копирайт**:
  - `copyrightText` / `copyrightTextRu`
  - год обычно подставляется автоматически на фронте.
- **Pop-up страницы**:
  - «Політика конфіденційності»:
    - `privacyPolicyText` / `privacyPolicyTextRu` (rich‑text);
  - «Повернення та обмін товарів»:
    - `termsText` / `termsTextRu` (rich‑text).

### Отображение на лендинге

Компонент футера (`components/landing/landing-footer.tsx`) и связанные модальные компоненты:

- Отображают:
  - название компании;
  - контакты (телефон, e‑mail);
  - иконки/ссылки на соцсети;
  - копирайт с текущим годом;
  - ссылки «Політика конфіденційності», «Повернення та обмін» и др.
- При клике на ссылки политик:
  - открывается модальное окно с содержимым `privacyPolicyText` или `termsText` (в зависимости от выбранного языка).

### Edge cases и рекомендации

- **Мультиязычность**:
  - если RU‑версии текстов нет, используется украинская;
  - для юридических текстов лучше явно поддерживать обе версии.
- **Пустые поля**:
  - если какой‑то контакт или ссылка не заданы — соответствующий элемент не отображается;
  - если не заполнены тексты политик — ссылки на pop‑up лучше скрывать или показывать дефолтный текст.
- **Форматирование HTML**:
  - юридические и политические тексты хранятся как HTML (`LongText`), поэтому:
    - требуется осторожность при вставке (не вставлять произвольные скрипты),
    - лучше использовать проверенные шаблоны/редакторы.
- **Многолендинговая консистентность**:
  - если один владелец использует несколько лендингов, возможны расхождения в юридическом тексте между ними;
  - для поддержания консистентности можно:
    - либо вынести глобальные настройки компании в отдельную таблицу/панель,
    - либо договориться обновлять все лендинги при изменении реквизитов.
