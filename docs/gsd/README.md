## Goal
Собрать в одном месте оглавление GSD‑документации по проекту Landing Conveyor и кратко описать назначение каждого документа.

## Solution
В каталоге `docs/gsd/` лежит набор файлов формата GSD (Goal → Solution → Details), каждый из которых отвечает за отдельный модуль системы: от общего обзора до конкретных подсистем (лендинги, варианты, форма заказа, диспетчер заявок, leads, видео, секции страницы, FAQ, футер, auth, деплой).

## Details

### Оглавление GSD‑документов

- `00-system-overview.md` — **Обзор системы**
  - Описывает, что такое Landing Conveyor, для кого он создан, какую бизнес-проблему решает, архитектуру (Admin UI → Backend API → DB + внешние CRM), стек и основные ENV‑переменные.

- `01-landings.md` — **Модуль управления лендингами**
  - Раскрывает модель `Landing`, CRUD‑маршруты `/api/admin/landings*`, slug‑маршрутизацию и статусы `draft/published`, а также настройки канала доставки заявок (`orderDestination`).

- `02-product-variants.md` — **Модуль вариантов товара**
  - Описывает сущность `Variant`, её связь с `Landing`, вложенные блоки (галерея, видео, преимущества, характеристики, таблицы размеров, отзывы), поля для интеграций (`cscartProductId`, `keycrmOfferSku`) и соответствующие Admin/API‑маршруты.

- `03-order-form.md` — **Модуль формы заказа**
  - Поясняет конфигурацию формы через `Landing.formConfig`, структуру полей (`name`, `phone`, `city`, `comment`), жизненный цикл заявки: `submit → валидация → Lead → отправка в CRM → статус` и endpoint `POST /api/leads`.

- `04-order-dispatch.md` — **Модуль диспетчеризации заявок**
  - Ключевой документ: описывает паттерн Adapter/Strategy, работу `OrderDispatcher`, адаптеры `CsCartAdapter` и `KeyCrmAdapter`, формат payload‑ов, обработку ошибок, retry‑логику и схему хранения ответов CRM.

- `05-leads.md` — **Модуль заявок (Leads)**
  - Описывает таблицу `Lead`, статусы `new/sent/failed`, API `/api/admin/leads*`, публичный retry‑маршрут `/api/leads/retry`, отображение заявок в админке и edge‑кейсы (массовый ретрай, удаление лендингов/вариантов).

- `06-video-storage.md` — **Модуль управления видео**
  - Разбирает процесс загрузки видео через `/api/admin/videos/upload`, хранение файлов в `public/video`/`public/uploads`, использование видео в вариантах (`videoUrl*`, `videoHtml*`), поддерживаемые форматы и ограничения по размеру.

- `07-page-sections.md` — **Модуль секций страницы**
  - Описывает JSON‑структуры секций «Як замовити» (`howToOrder`) и «Доставка та оплата` (`delivery`), редактирование в админке, мультиязычность и условную видимость секций на лендинге.

- `08-faq.md` — **Модуль FAQ**
  - Покрывает таблицу `FAQ` (вопросы-ответы с мультиязычностью и сортировкой), admin API `/api/admin/faqs*`, компонент `FAQEditor` и отображение FAQ аккордеоном на лендинге.

- `09-company-footer.md` — **Модуль инфо компании и футера**
  - Описывает поля `companyName`, `legalText`, контакты, соцсети, копирайт и HTML‑контент pop-up страниц (политика конфиденциальности, возврат), а также их использование во футере и модальных окнах лендинга.

- `10-auth.md` — **Модуль аутентификации**
  - Объясняет устройство таблицы `User`, работу `/api/auth/login`, `/api/auth/logout`, `/api/auth/me`, хранение JWT в httpOnly‑cookie, функцию `verifyAuth` и защиту admin‑маршрутов.

- `11-cloning-and-deployment.md` — **Деплой и клонирование**
  - Даёт чеклист по развёртыванию нового экземпляра: настройка `.env`, создание БД, миграции, сборка и запуск (npm/pm2/Docker), конфигурация Nginx/HTTPS и отличия между инстансами (домены, БД, CRM‑ключи).

### Как пользоваться GSD‑документацией

- Если нужно понять **общую картину системы** — начните с `00-system-overview.md`.
- Если вы работаете над конкретным модулем:
  - лендинги → `01-landings.md`;
  - варианты товара → `02-product-variants.md`;
  - форма заявки → `03-order-form.md` + `05-leads.md`;
  - интеграции с CRM → `04-order-dispatch.md` + соответствующие разделы `01/02/03`;
  - видео → `06-video-storage.md`;
  - секции страницы, FAQ, футер → `07/08/09`;
  - аутентификация и доступ → `10-auth.md`;
  - деплой/клонирование инстансов → `11-cloning-and-deployment.md`.
