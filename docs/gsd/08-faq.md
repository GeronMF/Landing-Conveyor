## Goal
Описать модуль FAQ: как задаются и хранятся вопросы-ответы для лендинга, как устроена мультиязычность и флаг «открыт по умолчанию», и как всё это отображается на лендинге.

## Solution
FAQ реализован как отдельная сущность `FAQ`, связанная с `Landing` отношением 1:N. Каждая запись содержит тексты вопроса и ответа на украинском и русском, порядок отображения и флаг `isOpen`, определяющий состояние аккордеона по умолчанию. CRUD для FAQ доступен через `app/api/admin/faqs*`, а админский UI — через компонент `FAQEditor` на вкладке «FAQ» в `/admin/landings/[id]`. На лендинге FAQ отображается аккордеон-списком с мультиязычной поддержкой.

## Details

### Схема БД: таблица `FAQ`

Фрагмент `schema.prisma`:

```text
model FAQ {
  id        String @id @default(uuid())
  landingId String

  question  String @db.Text
  questionRu String? @db.Text
  answer    String @db.Text
  answerRu  String? @db.Text

  order     Int    @default(0)
  isOpen    Boolean @default(false)

  landing Landing @relation("oldFaqs", fields: [landingId], references: [id], onDelete: Cascade)

  @@index([landingId])
  @@index([order])
}
```

### API‑маршруты FAQ

Маршруты находятся под `app/api/admin/faqs*` и/или `app/api/admin/landings/[id]/faqs`.

- **GET `/api/admin/landings/[id]/faqs`**
  - Назначение: получить список FAQ для лендинга, отсортированный по `order`.

- **POST `/api/admin/landings/[id]/faqs`**
  - Назначение: создать новый FAQ для лендинга.
  - Тело:
    - `question`, `questionRu?`, `answer`, `answerRu?`, `order?`, `isOpen?`.

- **PUT `/api/admin/faqs/[id]`**
  - Назначение: обновить FAQ.
  - Тело:
    - те же поля, что при создании, все опциональные.

- **DELETE `/api/admin/faqs/[id]`**
  - Назначение: удалить FAQ‑запись.

Админский компонент `FAQEditor` работает поверх этих маршрутов, предоставляя UI для добавления, редактирования, сортировки и удаления вопросов.

### Отображение на лендинге

- Компонент `FaqSection` (`components/landing/faq-section.tsx`) выполняет:
  - загрузку FAQ для текущего лендинга;
  - сортировку по `order`;
  - выбор языка (UK/RU) и fallback, если перевод отсутствует;
  - рендеринг аккордеона:
    - для каждого FAQ‑элемента:
      - заголовок — `question`/`questionRu`;
      - содержимое — `answer`/`answerRu`;
    - флаг `isOpen` определяет, открыт ли элемент по умолчанию.

### Edge cases и рекомендации

- **Мультиязычность**:
  - если `questionRu`/`answerRu` не заданы, используются украинские тексты;
  - важно, чтобы админ понимал, какой язык является базовым и как работают fallback‑ы.
- **Порядок отображения**:
  - `order` определяет порядок; рекомендуется:
    - всегда задавать его последовательно (1,2,3,...),
    - при удалении/добавлении пересчитывать порядок, чтобы избежать больших дыр.
- **Удаление лендинга**:
  - при удалении `Landing` его FAQ‑записи удаляются каскадно (onDelete: Cascade).
- **Очень длинные ответы**:
  - UI должен аккуратно работать с большими текстами (scroll, перенос строк, форматирование).
- **SEO / дублирование контента**:
  - FAQ может дублировать тексты на нескольких лендингах; если это важно для SEO, стоит продумать переиспользование блоков или canonical‑политику.
