## Goal
Описать модуль секций страницы («Як замовити» и «Доставка та оплата»): как настраиваются эти информационные блоки в админке и как они рендерятся на лендинге с учётом мультиязычности и условной видимости.

## Solution
Обе секции хранятся как JSON‑поля в модели `Landing` (`howToOrder` и `delivery`) и редактируются на вкладке «Секції сторінки» в админке. Компоненты лендинга (`HowToOrderSection`, `DeliverySection`) читают эти JSON‑структуры и рендерят соответствующие блоки. Если данные для секции пустые, она автоматически скрывается.

## Details

### Структура данных в БД

Фрагмент `Landing` в `schema.prisma`:

```text
model Landing {
  ...
  // Секція "Як замовити"
  howToOrder   Json? // {title, titleRu, productName, productNameRu, steps: [{icon, label, labelRu, text, textRu}]}
  // Секція "Доставка та оплата"
  delivery     Json? // {title, titleRu, shippingText, shippingTextRu, carriers: [{name, logoUrl}], promoText, promoTextRu, paymentNote, paymentNoteRu}
  ...
}
```

#### Секция «Як замовити» (`howToOrder`)

Структура:

```json
{
  "title": "Як замовити",
  "titleRu": "Как заказать",
  "productName": "Зимовий костюм",
  "productNameRu": "Зимний костюм",
  "steps": [
    {
      "icon": "clipboard",         // тип иконки
      "label": "Крок 1:",
      "labelRu": "Шаг 1:",
      "text": "Залиште заявку на сайті",
      "textRu": "Оставьте заявку на сайте"
    }
  ]
}
```

#### Секция «Доставка та оплата» (`delivery`)

Структура:

```json
{
  "title": "Доставка та оплата",
  "titleRu": "Доставка и оплата",
  "shippingText": "Ви обираєте поштовий сервіс на свій вибір",
  "shippingTextRu": "Вы выбираете почтовый сервис на свой выбор",
  "carriers": [
    { "name": "Нова Пошта", "logoUrl": "https://..." },
    { "name": "Укрпошта", "logoUrl": "https://..." }
  ],
  "promoText": "Оплачуйте доставку дешевше",
  "promoTextRu": "Оплачивайте доставку дешевле",
  "paymentNote": "Оплата при отриманні без передоплати",
  "paymentNoteRu": "Оплата при получении без предоплаты"
}
```

### API и Admin UI

Редактирование секций происходит через:

- `GET /api/admin/landings/[id]` — возвращает текущие значения `howToOrder` и `delivery`.
- `PUT /api/admin/landings/[id]` — принимает обновлённые JSON‑значения (валидируются как `z.any()` в текущей схеме, но структура предполагается как описано выше).

В админке `/admin/landings/[id]`:

- вкладка **«Секції сторінки»** содержит два блока:
  - «Як замовити?» — с формами для заголовка, названия товара и списка шагов;
  - «Доставка та оплата» — с полями для заголовка, описания доставки, списка перевозчиков, промо‑текста и условий оплаты.
- Для шагов используется селект иконок (например, `clipboard`, `phone`, `truck`, и т.д.).
- Для перевозчиков задаются `name` и `logoUrl`.

### Рендеринг на лендинге

Компоненты (см. `components/landing/how-to-order-section.tsx`, `components/landing/delivery-section.tsx`) выполняют:

- чтение `landing.howToOrder` и `landing.delivery`;
- выбор языка (UK/RU) и fallback на украинскую версию при отсутствии перевода;
- условный рендеринг:
  - если нет заголовка и/или шагов — секция «Як замовити» не показывается;
  - если нет заголовка и основного текста — секция «Доставка та оплата» не показывается;
  - список шагов и перевозчиков рендерится только если в массиве есть элементы.

### Edge cases и рекомендации

- **Пустые/частично заполненные структуры**:
  - если `howToOrder` существует, но `steps` пустой массив — компонент должен скрыть список шагов и/или всю секцию;
  - аналогично для `delivery` при пустом `carriers`.
- **Мультиязычность**:
  - если для RU‑поля нет значения, используется UK‑значение как fallback;
  - желательно избегать ситуаций, когда только один язык заполнен для заголовков.
- **Иконки шагов**:
  - значение `icon` должно быть из фиксированного списка, иначе компонент может отрисовать дефолтную иконку;
  - рекомендуется валидировать/ограничивать выбор в админке.
- **Ссылки на логотипы перевозчиков**:
  - `logoUrl` может быть пустым; в этом случае в UI просто отображается текст без логотипа;
  - желательно использовать только HTTPS‑ссылки на доступные изображения.
- **Совместимость с legacy‑данными**:
  - при миграции старых лендингов возможны структуры JSON, отличающиеся от описанных; компоненты должны быть устойчивыми к отсутствию некоторых полей (проверки на `undefined`).
